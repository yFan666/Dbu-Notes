# 知识库工作记录模板

> 来源：Second Brain（02 Knowledge/Templates/知识库工作记录模板规则.md）
> 整合日期：2026-06-18

---

## 用途

这个模板用于记录、整理和沉淀日常工作与知识库内容。

收到工作内容、今日计划、临时事项、会议信息、问题思考、资料摘录或粗糙笔记时，先判断内容类型，再按对应模板处理。

## 通用处理规则

1. 先判断内容类型：工作日志、日常事项、笔记沉淀、计划、复盘、待办或资料整理。
2. 如果没有明确要求总结，只进行记录，不主动扩写成总结。
3. 同一业务目标下的多个动作要合并为一个事项或需求，不要拆散。
4. 标题不要绑定项目名，项目名放到具体条目里。
5. 内容要简洁、可追溯，保留必要背景、目标、处理结果和后续动作。
6. 不确定分类时，先归到"待确认"，不要强行编造。
7. 需要输出时，默认输出一键复制 Markdown。

## 处理日志模板

适用：完成需求、修 BUG、调整逻辑、排查问题、代码审查或验证。

```md
### 需求或问题标题
- 项目：
- 类型：需求 / BUG / 排查 / 优化 / 审查
- 位置：
- 背景：
- 目标：
- 处理结果：
- 验证：
- 后续：
```

要求：同一目标下的拖拽拦截、提交兜底、展示校验等实现点合并为一个条目，不拆成多个需求。

## 日常事项模板

适用：今日计划、临时待办、会议安排、沟通事项、环境/账号/发布/流程类。

```md
### 事项标题
- 分类：计划 / 待办 / 会议 / 沟通 / 流程 / 发布
- 项目：
- 内容：
- 时间：
- 状态：待处理 / 进行中 / 已完成 / 暂停 / 待确认
- 下一步：
```

## 笔记记录模板

适用：技术思考、方案讨论、资料摘录、阅读笔记、决策依据、经验教训。

```md
### 笔记标题
- 分类：技术笔记 / 方案笔记 / 阅读摘录 / 决策记录 / 经验复盘
- 主题：
- 来源：
- 核心观点：
- 关键细节：
- 我的结论：
- 可复用规则：
- 后续动作：
```

要求：区分"原始资料"和"我的结论"；资料摘录不要改写成事实结论；决策记录保留当时背景、取舍和原因。

## 汇总输出模板

当要求"总结/汇总/日报/周报/月报"时：

```md
# YYYY-MM-DD 工作记录

## 今日计划
（事项标题 + 项目 + 内容 + 状态 + 下一步）

## 今日处理内容
（需求标题 + 项目 + 类型 + 位置 + 背景 + 目标 + 处理结果 + 验证 + 后续）

## 今日笔记
（笔记标题 + 分类 + 主题 + 核心观点 + 我的结论 + 可复用规则）

## 待确认

## 备注
```

## 合并示例

同一目标同时做了拖拽拦截和提交兜底，不拆成两个需求：

```md
### 1. 中医诊断不能位于第一行
- 项目：smarthos-front
- 类型：BUG
- 处理结果：
  - 拖拽时校验最终排序，第一条为中医诊断则拦截并回滚。
  - 提交时对有效诊断列表做兜底校验，第一条为中医诊断则阻止提交。
```
