# Codex 高频协作规则与提问沉淀

> 来源：Second Brain（00 Inbox/2026-06-17 最近10天高频规则与提问沉淀）
> 日期：2026-06-17
> 用途：后续封装项目专门 skill 的原始材料

---

## 结论

最近高频需求不是"让 AI 多做"，而是让 AI 更像一个**稳定的项目协作执行器**：少猜、少扩展、按证据闭环、能跨线程接续。

最值得封装成 skill 的不是某个具体项目知识，而是 4 类工作流：

1. 需求实现前的范围锁定
2. 参考页面 / 文档驱动实现
3. 防捏造与证据校验
4. 跨线程记录、继续与每日复盘

## 七条高频规则

### 1. 最小范围修改
> 默认只改和当前需求直接相关的内容。除非用户明确要求，不做顺手重构、不抽象公共组件、不扩大影响范围。

### 2. 不确定先问，不要擅自决定
> 证据不足时停止猜测。必须说明缺哪类证据，并给出最小补充信息；不能用"看起来合理"替代项目事实。

### 3. 参考页面 / 文档优先
> 实现前先找参考源，明确"照搬什么、差异是什么、哪些不能照搬"。参考源优先级：需求文档 > 指定页面 > 同模块相似页面 > 项目通用写法。

### 4. 验证闭环
> 完成后必须回到需求原文逐项对照：功能是否闭环、字段是否一致、权限/打印/导出是否有来源、样式是否破坏布局。

### 5. 记忆边界：全局只放协作偏好，项目内容进项目
> 全局长期记忆只保存跨项目稳定有效的回复纪律、协作偏好和工作方式。项目页面、组件、bug、接口、样式细节只进入项目上下文或项目记录。

### 6. 跨线程接续与阶段记录
> 跨线程总结必须读取当前线程、相关项目会话、当天文件变更、项目 git 记录和知识库记录；不能只凭当前对话判断。

### 7. UI 工作流：先定方向，再实现
> UI 不明确时，先用问题收敛风格，再输出可实现描述或设计图；用户确认后再改代码。视觉方向确认前，不进入实现。

## 高频提问模板

### A. 业务功能实现类
```text
处理这个需求：

目标文件：
参考页面/文档：
需求点：
限制：最小范围修改，不要擅自扩展；不确定先问。
完成后检查：字段、接口、权限、打印/导出、布局、是否有捏造逻辑。
```

### B. 需求整理 / 丢给研发类
```text
帮我把这个需求整理成研发可执行文档：

原始材料：
目标页面：
业务目标：
字段/接口/交互：
不确定项：请单独列出，不要替我脑补。
输出：研发任务清单 + 验收标准。
```

### C. 问题排查类
```text
帮我排查这个问题：

现象：
出现条件：
已知限制：是否能本地复现 / 是否只能线上出现
相关文件：
我希望输出：按可能性排序，先给最可能原因和最小验证办法。
```

### D. 规则 / 记忆 / Skill 类
```text
帮我处理一条协作规则：

规则内容：
生效范围：全局 / 当前项目 / 当前线程
是否写入知识库：是 / 否
是否后续封装成 skill：是 / 否
```

### E. UI / 动效类
```text
帮我处理 UI/动效方向：

目标感受：
参考图/页面：
必须保留：
不要出现：
当前阶段：只分析 / 出方案 / 出图 / 改代码
确认后再实现。
```

## 可封装的 4 个 Skill 草案

| 优先级 | Skill | 用途 |
|---|---|---|
| 1 | `anti-hallucination-implementation` | 防止捏造权限、接口、字段、业务逻辑。没有来源的逻辑不写，参考页面没有就标记待确认，完成后列出"证据来源" |
| 2 | `project-requirement-executor` | 处理业务页面需求：读取目标文件/需求/参考页面→列出照搬点和差异点→最小范围实现→对照需求逐项验证 |
| 3 | `daily-work-review` | 每日事项整理和跨线程接续：查当前线程→查最近会话→查项目 git 和文件变更→按项目归类→输出总结 |
| 4 | `ui-direction-to-implementation` | 把模糊 UI 感受转成可实现方案：先问风格问题→提炼视觉关键词→输出可编程描述→必要时出图 |

**原因**：最近最影响效率的不是能力不足，而是"证据边界"和"执行范围"需要更稳定。

## 可直接进入项目 AGENTS 的短规则

```markdown
## 执行边界

- 默认最小范围修改，只处理当前需求直接相关内容。
- 参考页面、需求文档、用户给出的字段和接口优先；没有来源的逻辑不要捏造。
- 不确定时先问，不能擅自决定。
- 完成后按需求逐项检查：字段、接口、权限、打印/导出、布局、验证结果。
- 项目细节写入项目上下文或知识库，不进入全局长期记忆。
```
