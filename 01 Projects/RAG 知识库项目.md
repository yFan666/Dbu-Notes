# RAG 知识库项目

> 项目：个人 RAG 知识库（围绕成长、项目实践、想法管理）
> 来源：整合自 Second Brain（项目计划 + PROJECT_CONTEXT）
> 整合日期：2026-06-17

---

## 项目目标

构建一个围绕个人成长、项目实践和想法管理的 RAG 知识库。

一句话目标：把知识库建设成一个**可被 AI/RAG 使用的个人系统**，用来管理项目、整理想法、沉淀经验，并辅助长期成长。

需要帮助完成三件事：

1. 管理和推进项目
2. 捕获、整理和转化自己的小想法
3. 沉淀问题、解决办法、踩坑经验和成长路径

## 真正想解决的问题

不是单纯做"能搜索文档的工具"，而是建立可持续复利的个人知识系统：

- 项目做完后，经验不要丢
- 小想法不要散落在聊天、脑子或临时文件里
- 遇到的问题、走过的弯路、解决办法要沉淀下来
- AI 不只是回答问题，而是能基于知识库帮你继续推进

## 项目原则

1. **先让知识库本身变得清晰，再做 RAG**
2. **RAG 的效果取决于文档质量，不只取决于模型或框架**
3. 每个项目都应该有自己的 `PROJECT_CONTEXT.md`
4. 重要项目过程要记录问题、决策、失败原因和复盘
5. 想法先捕获，再筛选，不在一开始就过度分类

## 待确认问题

- RAG 最终希望运行在哪里：本地、云端，还是 Obsidian 插件内
- 知识库是否需要接入代码项目、禅道、日报等外部数据
- 是否需要区分私人资料和可公开资料
- 是否需要为每篇笔记增加统一 metadata

## 阶段计划（5 阶段）

### 阶段 1：整理知识结构
目标：让现有 Markdown 文件变成可管理、可检索、可持续更新的资料。

要做：
- 建立 `AI_CONTEXT.md` 作为跨工具入口
- 建立 `AGENTS.md` 作为 Agent 协作规则
- 建立 `00 Inbox/想法` 用来捕获想法
- 建立项目级 `PROJECT_CONTEXT.md`
- 明确每类内容应该放在哪里

完成标准：新 AI 工具进入知识库后知道先读什么；新想法知道先放哪里；新项目知道怎么建上下文。

### 阶段 2：建立想法流转
目标：让想法从"随手记录"变成"可判断、可执行、可沉淀"。

```
捕获想法 -> 00 Inbox/想法/待处理想法.md -> 每周整理 -> 转为项目/长期知识/外部资料/归档
```

想法判断标准：
- 近期要做 → `01 Projects`
- 可复用认知 → `02 Knowledge`
- 只是资料入口 → `02 Knowledge/Resources`
- 暂不做 → `99 Archive`

完成标准：每条想法都有状态；每周至少清理一次 Inbox；重要想法不会长期停留在 Inbox。

### 阶段 3：建立项目沉淀机制
目标：每个项目都能积累上下文，而不是只留下最终结果。

每个项目建议包含：
```
PROJECT_CONTEXT.md   项目目标、当前状态、关键决策
任务记录.md           编号、日期、模块、状态
问题与决策.md         遇到的问题、解决办法
复盘.md              经验回流
```

完成标准：AI 接手项目时可先读 `PROJECT_CONTEXT.md`；项目经验能回流到 `02 Knowledge`。

### 阶段 4：设计 RAG 最小可行版本
目标：让 AI 能基于 Markdown 知识库回答问题，而不是只靠聊天上下文。

最小版本能力：
- 读取指定目录下的 Markdown
- 按语义或关键词检索相关片段
- 基于检索内容回答
- 回答时引用来源文件
- 找不到答案时明确说明信息不足

优先支持的问题：我现在有哪些想法可推进？某项目当前进展？之前遇到过类似问题吗？某技术点整理过哪些资料？下一步该做什么？

完成标准：能对 `00 Inbox`、`01 Projects`、`02 Knowledge` 做基础问答；回答带来源；不编造知识库里没有的信息。

### 阶段 5：接入 Agent 自动化
目标：让 RAG 不只是问答，还能辅助执行任务。

优先候选 Agent：
1. 知识库问答 Agent（风险最低，先做）
2. Inbox 整理 Agent（最贴合当前需求）
3. 自动日报 Agent（涉及外部系统，后置）
4. 前端代码审核 Agent（涉及代码质量标准，后置）
5. 项目复盘 Agent

## 30 天计划

| 周 | 重点 |
|---|---|
| 第 1 周：结构稳定 | 完成 `00 Inbox/想法`；建立 RAG 项目上下文；建立项目沉淀模板；整理已有想法 |
| 第 2 周：内容标准化 | 设计项目/知识/想法笔记模板；整理 3-5 篇高质量样例笔记 |
| 第 3 周：RAG 最小原型 | 选本地 Markdown 目录作数据源；实现基础检索；实现基于来源的回答；测试 10 个真实问题 |
| 第 4 周：Agent 化 | 做知识库问答 Agent；做 Inbox 整理 Agent 草稿；记录失败案例；更新工作规则和项目上下文 |

## 当前优先级

**优先做**：完善 `00 Inbox/想法`。

**暂时不急**（等结构和文档质量先稳定）：
- 复杂向量数据库
- 多知识库架构
- 自动提交日报
- 全自动代码审核
