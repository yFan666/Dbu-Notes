# PM 需求文档模板

> 来源：Second Brain（02-Knowledge/04-Prompt/PM.md + 00-Inbox/项目 Prompt.md）
> 整合日期：2026-06-18

---

> 最终目标：实现后端 router 和前端页面的关联——一个需求描述，接口和页面关联实现。

## 需求书写结构

- **开发前**：基础信息、需求内容、验收标准
- **开发后**：代码定位、实现改动、验证结果、遗留问题

> 最重要的是补上**验收标准**，它能让 PM、研发和 AI 对"需求是否完成"形成统一判断。

---

## 模板

```markdown
# 页面名称 - 需求关键词

## 一、页面信息

### 1. 页面名称
> 例如：【xxx页面】

### 2. 子应用名称
> 例如：项目名称，或者 monorepo 的子应用名称

### 3. 文件路径
> 例如：该项目下的相对路径

## 二、需求内容

### 1. 需求目标
简要说明本次需求要解决的问题及最终效果。

### 2. 功能说明
- 新增功能：
- 修改功能：
- 删除功能：

### 3. 关联接口
- 查询接口：
- 保存接口：
- 其他接口：

### 4. 查询条件
没有就不写。
{
  查询条件名称：{ 字段名称, 控件类型, 数据源来源 }
}

### 5. 列表字段
没有就不写。
{
  列名：字段名称
}

### 6. 交互与业务规则
1. 用户执行什么操作。
2. 系统产生什么结果。
3. 特殊值、边界条件及异常情况如何处理。

### 7. 验收标准
1. 页面功能可正常操作。
2. 查询、保存及分页结果符合预期。
3. 禁用、校验和异常提示符合业务规则。
4. 不影响原有功能。
```
