# Git 提交规则

## 触发场景

只在需要暂存、提交、推送、查看提交范围时读取本规则。

普通知识库整理、方案讨论、内容总结不需要读取。

## 提交前

提交前先查看当前改动，只提交和本次任务相关的文件。

如果工作区存在无关改动，不要默认一起提交。

## 提交信息

提交信息推荐格式：

```text
类型: 简短说明 颜文字
```

推荐类型：

- 新增
- 优化
- 规则
- 整理
- 归档
- 修复

示例：

```text
新增: 初始化知识库结构 (๑•̀ㅂ•́)و✧
优化: 整理 Inbox 临时记录 (｡•̀ᴗ-)✧
规则: 补充 Git 提交规范 (ง •̀_•́)ง
```
