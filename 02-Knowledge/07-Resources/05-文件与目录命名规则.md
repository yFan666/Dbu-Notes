# 文件与目录命名规则

## 触发场景

- 新建文件或目录。
- 重命名文件或目录。
- 批量调整知识库路径。
- 修改总览、规则入口或 Obsidian 工作区路径。

## 命名标准

- 文件名和目录名不使用空格或 `_`，统一使用 `-` 分隔。
- 连续多个空格或 `_` 压缩成一个 `-`。
- 保留中文标点、中文括号、日期格式、英文大小写和扩展名。
- 正文标题使用自然语言，不需要跟随路径加入 `-`。
- 工作区根目录 `C:\Dbu Notes` 暂时保留，不纳入本规则的自动改名范围。

示例：

```text
00 Inbox -> 00-Inbox
03 Dbu_Notes -> 03-Dbu-Notes
00 项目说明.md -> 00-项目说明.md
00_01_2026.06.22-2026.06.28 日常工作.md
-> 00-01-2026.06.22-2026.06.28-日常工作.md
```

## 链接规则

- Wiki 链接优先写完整的库内相对路径。
- 链接显示文字使用正文标题或自然语言别名。
- 存在同名文件时，必须使用完整路径，不能只写文件名。
- 附件链接使用 `assets` 下的完整相对路径。
- 重命名后必须同步更新总览、AGENTS、规则文档、正文路径和 `.obsidian/workspace.json`。

示例：

```markdown
[[02-Knowledge/01-知识库最小闭环规则|知识库最小闭环规则]]
[[01-Projects/01-smarthos-cdms-manager/00-项目说明|smarthos-cdms-manager]]
![[assets/Pasted-image-20260611144758.png]]
```

## 批量重命名流程

1. 生成旧路径到新路径的完整映射。
2. 检查重名、覆盖和工作区外路径。
3. 按路径深度从深到浅执行重命名。
4. 更新 Markdown、总览、规则入口和 Obsidian 工作区引用。
5. 检查文件数量、失效链接、附件和 JSON。

隐藏系统目录如 `.git`、`.obsidian`、`.agents` 不重命名；只在必要时更新其中的内容路径引用。

## 验收标准

- 内容区文件名和目录名中不存在空格或 `_`。
- 重命名前后文件和目录数量一致。
- 总览和 AGENTS 中的入口目标存在。
- 本次重命名不新增失效 Wiki 链接。
- 重复文件名均通过完整路径引用。
- 附件可解析，`.obsidian/workspace.json` 是有效 JSON。
