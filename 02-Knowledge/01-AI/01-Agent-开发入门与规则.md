# Agent 开发入门与规则

> 来源：Second Brain（02-Knowledge/01-AI/Agent Development）
> 整合日期：2026-06-17
> 定位：Agent 开发的长期知识、实践规则和项目复盘沉淀

---

## 第一阶段学习目标

第一阶段不要急着追框架，先把 Agent 的工作方式跑通。

你需要先理解：

- Agent 不是单次问答，而是一个**可以循环执行任务的系统**
- Agent 的核心能力来自：模型推理、工具调用、上下文管理、状态记忆、结果评估
- 一个可靠 Agent 的关键不是"更聪明"，而是**"边界清楚、步骤可控、错误可恢复"**

第一阶段建议做 3 个小练习：

1. **工具调用 Agent**：让 Agent 根据用户目标决定是否调用一个本地函数
2. **知识库问答 Agent**：让 Agent 只基于指定文档回答，并能引用来源
3. **任务执行 Agent**：让 Agent 拆解任务、执行步骤、检查结果、输出总结

## 必须理解的核心知识

### 1. Agent 基础概念

- **LLM**：负责理解、推理和生成
- **Prompt**：定义角色、目标、约束和输出格式
- **Tool**：Agent 可调用的外部能力（搜索、文件读写、数据库、API）
- **Memory**：保存短期上下文和长期偏好
- **Planning**：把复杂目标拆成步骤
- **Reflection**：检查结果是否满足目标
- **Guardrails**：限制 Agent 的权限和行为边界

### 2. 工具调用

工具调用是 Agent 开发的核心。需要重点理解：

- 什么时候应该调用工具
- 工具输入输出如何结构化
- 工具失败时如何重试或降级
- 哪些工具需要权限确认
- 工具返回的信息如何进入下一轮推理

### 3. 上下文工程

Agent 的质量高度依赖上下文：

- 哪些信息应该放进系统规则
- 哪些信息应该作为用户任务输入
- 哪些信息应该通过检索动态加入
- 哪些历史信息应该被总结，而不是完整塞入上下文
- 如何避免无关信息污染判断

### 4. 记忆系统

记忆不是把所有内容都保存下来。建议分为：

- **用户偏好**：长期稳定的习惯和要求
- **项目状态**：当前项目的目标、约束、进度
- **知识沉淀**：可复用的概念、方法、案例
- **操作记录**：关键决策、失败原因、验证结果

### 5. 评估与调试

Agent 必须可评估，否则很难迭代。初期至少记录：

- 输入是什么
- Agent 计划是什么
- 实际调用了什么工具
- 输出是否正确
- 错误发生在哪一步
- 下次如何改规则或工具

## 初始化开发规则

### 总原则

1. 先定义任务边界，再设计 Agent
2. 先做最小可运行版本，再扩展复杂能力
3. 任何 Agent 都必须有明确输入、输出、工具权限和失败处理
4. Agent 的每一步都应该能被记录、复盘和改进
5. 不让 Agent 在没有确认的情况下执行高风险操作

### Prompt 规则

1. Prompt 必须说明目标、角色、边界、输出格式
2. 不把长期规则和临时任务混在一起
3. 对关键任务使用结构化输出（JSON、Markdown 表格、固定小节）
4. 避免只写"你是一个专业助手"，要写清楚"你能做什么、不能做什么、如何判断完成"
5. 每次改 Prompt 都要记录改动原因和效果

### 工具规则

1. 每个工具只做一件明确的事
2. 工具输入必须结构化，避免依赖自然语言猜测
3. 工具输出要尽量简洁，返回 Agent 下一步判断真正需要的信息
4. 涉及删除、覆盖、转账、发送消息、发布内容等操作时，必须先请求确认
5. 工具失败时要返回可理解的错误，而不是只返回失败

### 记忆规则

1. 不保存无价值的聊天流水
2. 只保存可复用、可验证、会影响未来决策的信息
3. 用户偏好、项目规则、技术决策分开保存
4. 记忆内容要注明来源和日期
5. 发现旧记忆失效时，不直接删除，先标记为过期或替换

### 实践规则

1. 每个 Agent 实验都要有一个独立笔记
2. 每个实验记录目标、输入、工具、流程、结果、问题、下一步
3. 不一次做"大而全 Agent"，优先做能完成一个明确任务的小 Agent
4. 每次只改一个关键变量（Prompt、工具、记忆或评估方式）
5. 实验成功后，把经验提炼回这篇规则笔记

## 第一个实践项目建议

**项目名**：知识库问答 Agent

目标：
- 用户输入一个问题
- Agent 只从指定 Markdown 笔记中查找答案
- 找不到答案时明确说"当前知识库没有足够信息"
- 回答时给出引用的笔记名称

为什么先做这个：
- 和个人知识库天然结合
- 能练到检索、上下文、引用、边界控制
- 风险低、反馈快，后续可扩展成个人知识库助手
