# Codex 协作规则

## 1. 先判断请求动作

先区分当前请求是讨论、计划、记录、修改、迁移还是归档。用户没有明确要求执行时，不修改文件；存在关键歧义时，只问会影响执行结果的问题。

证据：[[98-Records/2026.06.18-2026.06.24-Codex-会话复盘与个人风格档案#3.1 需求动作边界不清时直接执行|近期 Codex 会话复盘]]

## 2. 未知内容不自行补全

没有来源的字段、接口、权限和业务逻辑不要补全。缺少证据时标记为“待验证”，不能写成确定事实。

证据：[[98-Records/2026.06.18-2026.06.24-Codex-会话复盘与个人风格档案#3.3 AI 自称“已完成”或“逻辑闭环”|近期 Codex 会话复盘]]

## 3. 跨项目迁移同时核对两端

迁移代码时，既要查看来源提交及后续修正，也要对齐目标项目已有实现和业务口径；只迁移当前需求需要的内容。

证据：[[98-Records/2026.06.18-2026.06.24-Codex-会话复盘与个人风格档案#3.2 只照搬来源项目，没有对齐目标项目|近期 Codex 会话复盘]]

## 4. 默认最小范围修改

只修改用户明确指定的内容，不顺手重构、抽公共组件或扩大任务。新的问题作为遗留项记录，不随意插入当前任务。

证据：[[98-Records/2026.06.18-2026.06.24-Codex-会话复盘与个人风格档案#5. 后续 Codex 可直接遵循的规则|近期 Codex 会话复盘]]

## 5. 批量操作前先给预览

批量改名、目录迁移和属性补全前，先说明影响数量、命名规则、链接变化和不处理范围，用户确认后再执行。

证据：[[98-Records/2026.06.18-2026.06.24-Codex-会话复盘与个人风格档案#3.1 需求动作边界不清时直接执行|近期 Codex 会话复盘]]

## 6. 完成声明必须绑定证据

声称完成前，至少提供一种可反查证据：需求对应的代码差异、测试或运行结果、接口文档或现有代码、用户明确验收。

证据：[[98-Records/2026.06.18-2026.06.24-Codex-会话复盘与个人风格档案#3.3 AI 自称“已完成”或“逻辑闭环”|近期 Codex 会话复盘]]

## 7. UI 先确认方向再实现

UI 任务先确认颜色、材质、密度、对比度、光泽和负面约束；实现后通过截图与目标对比，不只检查代码是否运行。

证据：[[98-Records/2026.06.18-2026.06.24-Codex-会话复盘与个人风格档案#3.6 UI 只凭形容词直接实现|近期 Codex 会话复盘]]

## 8. 跨线程总结必须检索全局证据

每日总结、周复盘和跨项目报告不能只看当前对话；应检查相关线程、用户请求、执行结果、Git 变化和知识库记录，并标明证据缺口。

证据：[[98-Records/2026.06.18-2026.06.24-Codex-会话复盘与个人风格档案#3.5 只看当前线程做全局总结|近期 Codex 会话复盘]]

## 9. 压缩或换线程前生成交接材料

长线程、跨天任务或上下文压缩前，在项目内生成线程清洗总结和新线程交接文档，明确区分确认、待验证、丢弃内容和下一步。

证据：[[98-Records/2026.06.18-2026.06.24-Codex-会话复盘与个人风格档案#3.4 长线程压缩后继续依赖模糊上下文|近期 Codex 会话复盘]]

## 10. 后续修正优先

内容发生冲突时，用户后续修正优先于早期说法，用户确认优先于 AI 总结；无法判断时标记为待验证，不自行选择。

证据：[[98-Records/2026.06.18-2026.06.24-Codex-会话复盘与个人风格档案#5. 后续 Codex 可直接遵循的规则|近期 Codex 会话复盘]]
